# KeeperFX 32位 (i686) 编译与二次开发日志

本日志记录了在 64 位 Windows 11 环境下，针对 KeeperFX 进行 32 位架构适配、启动器兼容性修复以及存档对齐尝试的全过程。

## 1. 编译环境配置 (2026-04-01 ~ 2026-04-02)
*   **工具链**: MinGW-w64 (i686-win32-dwarf-rev0, GCC 8.1.0)。
*   **目标架构**: 32nd bit (i686-w64-mingw32)。
*   **依赖库处理**: 
    *   通过 `objcopy --rename-section .rsrc=.rsrc_ignore` 对 `libSDL2.a` 执行资源段重命名。
    *   **原因**: 防止链接时 `libSDL2.a` 自带的图标/版本信息与主程序的 `VERSIONINFO (ID 1)` 发生冲突，确保生成的 `keeperfx.exe` 能被官方 Launcher-Qt 正确识别。

## 2. 核心功能改进
### 2.1 语音与文字语言独立配置 (Speech/Text Decoupling)
*   **变动**: 扩展 `InstallInfo` 结构体，新增 `speech_lang_id`。
*   **实现**: 
    *   在 `config_keeperfx.c` 中添加 `SPEECH_LANGUAGE` 配置命令。
    *   修改 `bflib_sndlib.cpp` 中的 `load_sound_banks`，使其优先从 `speech_lang_id` 对应的目录加载 `speech_*.dat`。
    *   **效果**: 允许玩家在使用中文文本的同时，保留英文原始语音。

### 2.2 启动器 (Launcher) 兼容性锁定
*   **修正点**: 将 `res/keeperfx_stdres.rc` 中的 `VERSIONINFO` 资源 ID 锁定为 **1**。
*   **稳定性**: 彻底解决了启动器报错“无法识别的执行文件”或无法保存设置的问题。

## 3. 存档兼容性探索 (4778 版本对齐)
*   **诊断发现**: 旧版 4778 存档的 `GameOrig` 数据块（51.39MB）与当前源码编译的版本（51.04MB）存在 **348,882 字节** 的断层。
*   **断层成分**: 
    *   约 346,800 字节对应于 $170 \times 170$ 地块的字段扩充（每个地块多出 12 字节逻辑数据）。
    *   约 2,082 字节对应于结构体末尾的调试快照冗余。
*   **结论**: 虽然通过 `SlabMap` 补全字段可以强制加载，但由于字段语义极度重叠（如地块所有权或光照缓存位置不明），加载后会导致内存越界与逻辑崩溃。
*   **最终方案**: 为了保证二次开发后的游戏逻辑纯净与稳定，**已决定放弃对 4778 版老存档的兼容**，回归 KeeperFX 官方主线逻辑基线。

## 4. 维护建议
*   后续开发应基于此 32 位稳定基线进行。
*   若需再次尝试存档对齐，需寻找 4778 版本的原始定义头文件。
